## Prevention Rule Reference

| Code | Rule |
|------|------|
| W01 | Single state read per check cycle — never read state twice in same function |
| W02 | Never fall back to SL price for non-SL exits — fetch mark price instead |
| W03 | Deep copy all position data returned from state manager |
| W04 | Always pass dict copies to retry calls |
| W05 | Filter orders by positionSide when attributing fills |
| W06 | Cache exchange metadata with TTL — never fetch per-trade |
| W07 | Name commission variables explicitly: `_rt` or `_per_side` suffix |
| W08 | Log warnings for symbols stuck at < 2 bars for 3+ cycles |
| W09 | Pass singletons explicitly — no module-level global state |
| W10 | Sort orders by timestamp, not by ID |
| W11 | Use TimedRotatingFileHandler(backupCount=30) for log rotation |
| W12 | Send Telegram on ALL state mutations that affect PnL |
| W13 | Throttle warmup requests — 100ms minimum |
| W14 | Use datetime.now(timezone.utc) everywhere |
| W15 | Use absolute paths: Path(__file__).resolve().parent |
| W16 | Consistent API contracts — helpers either always or never check error codes |
| W17 | No duplicate utility functions — shared helpers in api_utils.py |
| W18 | place-then-cancel SL replacement — never cancel-then-place |
| W19 | Always log tracebacks — exc_info=True on all logger.error for exceptions |
