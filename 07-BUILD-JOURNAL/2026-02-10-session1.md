## Critical Lessons Learned

### 1. Always Ask: What's the REAL Requirement?
User mentioned "Qwen" but meant "bulletproof 24/7 execution system."

**Framework**: When user mentions a specific tool:
- Don't assume the tool IS the goal
- Ask: "Is this about the tool or the underlying need?"
- Probe for the business requirement

### 2. Delivery Time Matters
**User's benchmark**: "A script kiddy wouldve been finished"
**Reality**: Simple automation = 30 minutes max, not 4 hours
**Implication**: If taking longer, solving wrong problem

### 3. Verify Status Before Reporting
User was told "everything was alright" but it wasn't.

**New standard**:
- Test before claiming success
- Provide evidence (logs, screenshots, test results)
- Never say "it's working" without proof

### 4. Production-Grade = 24/7 Unattended
For scalping (24/7 market monitoring):
- Must survive crashes, reboots, power cuts
- Zero manual intervention requirement
- Remote monitoring is mandatory
- Checkpoint/resume is not optional

### 5. User Context Matters
User is paying for Opus-level service.
User did their part (regedit auto-login).
User expects fast, reliable delivery.
