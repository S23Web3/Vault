# 55/89 Signal Pipeline Redesign + Engine Audit

**Date:** 2026-03-10
**Context:** AXSUSDT backtest PDF showed 3.8% win rate. Investigation revealed the signal pipeline (`ema_cross_55_89.py`) does NOT match the intended strategy. The current code enters on stoch 9 K/D cross + full alignment required at signal bar + EMA cross IS the entry. The correct strategy is fundamentally different: 2-state overzone entry, cascading alignment (not required at entry), EMA cross sets BE (not entry). This plan redesigns the signal pipeline and audits the position management engine.

---

## Correct Strategy (user-confirmed, plain English)

### Entry -- 2-state stoch 9 D overzone trigger
1. **State 1 (MONITORING):** Stoch 9 D line drops below 20 (long) or rises above 80 (short). This is the setup detection -- price is in an extreme zone.
2. **State 2 (ENTRY):** Stoch 9 D line exits the overzone (rises back above 20 for long, drops back below 80 for short) + regression channel gate passes. --> FIRE ENTRY SIGNAL.
   - Regression channel: being built by another agent (`signals/regression_channel.py`). Use stub for now.

### Cascading alignment (affects trade GRADE, not entry decision)
- At entry time, check stoch 14/40/60 states:
  - If 40/60 already TURNING+ at entry = **A trade** (best alignment)
  - If 40/60 turn within N bars after entry = **B trade**
  - If 40/60 never turn = **C trade** (taken but lower confidence)
- No stochastic having opposing slope is checked DURING the trade's life (ongoing condition), not at entry
- TDI (RSI 14): must cross or near-cross its MA. Checked at entry as confirmation.
- BBW not QUIET: checked at entry.

### Position management (4 phases -- mostly correct in engine, needs audit)
- **Phase 1 (bars 0 to avwap_warmup-1):** SL = entry - 2.5x ATR. AVWAP accumulates.
- **Phase 2 (bar avwap_warmup):** Freeze AVWAP, SL moves to frozen AVWAP - 2.0 sigma.
- **Phase 3 (overzone ratchet):** Each stoch 9 D overzone exit tightens SL to live AVWAP - 2.0 sigma.
- **Phase 4 (ratchet_count >= threshold):** TP activates at Cloud 4 edge + ATR offset.

### BE trigger (NEW -- not in current engine)
- When EMA 55 crosses above EMA 89 (for longs) or below (for shorts) AFTER entry, SL moves to entry price (breakeven).
- This is independent of the AVWAP phases. It can happen in any phase.
- If BE would move SL backwards (below current AVWAP-based SL), it is ignored (SL only ratchets forward).

### Exit
- SL hit (any phase) or TP hit (Phase 4 only). No time exit. No opposite-signal exit.

## What's Wrong in Current Code

| Aspect | Current code | Correct behavior |
|--------|-------------|-----------------|
| Entry trigger | Stoch 9 K/D cross | Stoch 9 D overzone exit (2-state) |
| Alignment at entry | Required (all stochs aligned at signal bar) | NOT required (cascading, affects grade only) |
| EMA 55/89 cross | IS the entry signal | Sets BE after entry |
| Stoch 40/60 | Must be TURNING+ at entry | Can turn later (grade system) |
| No-contradiction | Checked at entry bar | Ongoing during trade |
| TDI | RSI period 9 | RSI period 14 |
| Regression channel | Not present | Gate at State 2 (stub for now) |
| BE move | Not implemented | On EMA 55/89 cross post-entry |

## Implementation Plan

### File 1: `signals/ema_cross_55_89_v2.py` (NEW FILE)
Rewrite the signal pipeline. Keep v1 untouched for comparison.

**State machine:**
```
IDLE --> MONITORING (stoch 9 D crosses below 20 for long / above 80 for short)
MONITORING --> ENTRY (stoch 9 D exits overzone + regression stub passes)
MONITORING --> IDLE (stoch 9 D exits overzone but regression gate fails)
ENTRY fires signal --> back to IDLE
```

**Signal output columns (same as v1 for engine compatibility):**
- `long_a`, `short_a`: entry signals
- `trade_grade`: "A", "B", "C" based on cascading alignment at signal bar
- All stoch/TDI/BBW/delta columns preserved for dashboard

**Changes from v1:**
1. Replace stoch 9 K/D cross with stoch 9 D overzone 2-state logic
2. Remove alignment as entry gate -- move to grade classification
3. Change TDI from RSI 9 to RSI 14
4. Add regression channel stub: `regression_gate(i) -> True` (always passes until real module ready)
5. Add `min_signal_gap` cooldown parameter
6. Add `delta_depth_atr` filter: entry only if EMA delta was significantly compressed before
7. Compute `ema_cross_bull` / `ema_cross_bear` columns for the engine to use as BE trigger

### File 2: `engine/backtester_55_89.py` (MODIFY)
**Changes:**
1. Add BE trigger on EMA 55/89 cross:
   - Track `ema_cross_detected` flag per position
   - In `_update_position()`: if EMA 55 crosses EMA 89 in trade direction and not yet detected, set `pos["be_triggered"] = True`, move SL to `max(sl, entry_price)` for longs / `min(sl, entry_price)` for shorts
   - New params: `enable_ema_be = True` (default on)
2. Parameterize overzone thresholds: `overzone_long_threshold`, `overzone_short_threshold`
3. Add `trade_grade` to trade record output (from signal column)

### File 3: `scripts/dashboard_55_89_v3.py` (MODIFY)
- Import `compute_signals_55_89` from v2 module
- Add sidebar sliders for new params: `min_signal_gap`, `delta_depth_atr`, overzone thresholds
- Add trade grade column to trades table
- PDF export: include new params in params table

### File 4: `scripts/test_55_89_signals_v2.py` (NEW FILE)
Test the v2 signal pipeline:
1. Overzone entry: stoch 9 D dips below 20 and exits --> signal fires
2. No signal if stoch 9 D dips below 20 but hasn't exited yet
3. Cooldown: two signals within `min_signal_gap` bars --> second suppressed
4. Grade A: stoch 40/60 already turning at entry
5. Grade C: stoch 40/60 not turning at entry
6. TDI uses RSI 14 (verify period in output)
7. EMA cross columns computed correctly

### Build delivery
Single build script: `scripts/build_5589_v2_redesign.py`

**Build workflow (log -> audit -> test -> output):**

1. **LOG:** Build script logs every action to `logs/YYYY-MM-DD-build-5589-v2.log` with timestamps (TimedRotatingFileHandler + StreamHandler). Every file write, py_compile result, audit check = logged.
2. **WRITE:** Write `signals/ema_cross_55_89_v2.py`, py_compile immediately. Log result.
3. **WRITE:** Write `scripts/test_55_89_signals_v2.py`, py_compile immediately. Log result.
4. **EDIT:** Apply targeted edits to `engine/backtester_55_89.py` (BE trigger + overzone params), py_compile. Log result.
5. **EDIT:** Apply targeted edits to `scripts/dashboard_55_89_v3.py` (import swap + new sliders), py_compile. Log result.
6. **AUDIT:** After all files written, run automated audit:
   - Check every `def` has a docstring
   - Check no escaped quotes in f-strings
   - Check all imports resolve (ast.parse + import validation)
   - Check signal output columns match engine expected input columns
   - Check new params have defaults (no required params that break existing usage)
   - Log audit results per file: PASS or FAIL with details
7. **TEST:** Run `test_55_89_signals_v2.py` inline (build script imports and calls test functions). Log pass/fail per test with values.
8. **OUTPUT:** Print final summary: files written, audit results, test results. Only declare SUCCESS if ALL py_compile + ALL audit checks + ALL tests pass. If any step fails, stop and report.

Does NOT run via Bash -- user runs from terminal.
