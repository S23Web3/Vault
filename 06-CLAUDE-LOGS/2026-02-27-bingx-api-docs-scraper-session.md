## Runtime Debug Log (3 rounds of fixes)

### Round 1: Initial build -- test mode ran, 0 extraction
- Scraper navigated successfully, 3/3 pages visited, 0 failures
- But: `Extracted: 0 params, 0 response fields, 0 error codes` on all 3 pages
- Output was 917 lines of raw text dump (full_text as info pages)
- Root cause chain: `_detect_page_type()` checked for `"GET /"` but BingX puts method and path on separate lines (`GET\n/openApi/...`), so every page was classified as `"info"` and `_parse_rest_endpoint` was never called

### Round 2: Fixed page type detection -- still 0 extraction
- Added standalone method detection (`if line.strip() in ("GET", "POST", ...)`) and `/OPENAPI/` fallback
- STILL got 0/0/0 -- same output, same file size (15788 chars, 917 lines)
- Realized the user was running an older cached version (timestamps 10:25 vs edit at 10:27)
- User re-ran after the fix took effect

### Round 3: Paired table merge -- extraction working
- Discovered BingX DOM renders each table section as TWO `<table>` elements: a 1-row header table + a separate data table
- Added `_merge_paired_tables()` to detect and merge consecutive 1-row + N-row pairs
- Also fixed: `"filed"` typo (not `"field"`), `"msg"` (not `"message"`), trailing empty cells
- Result: `3 params, 20 response fields, 0 error codes` / `4 params, 5 response fields, 4 error codes` / `4 params, 5 response fields, 3 error codes`
- Output dropped to 631 lines (structured content, not raw text)
- Scroll-into-view fix also added for hidden menu items (Signature Auth timeout)
- Stale progress file cleanup added (test mode deletes leftover progress)

### All bugs found and fixed

| Bug | Root Cause | Fix |
|-----|-----------|-----|
| Page type always "info" | `_detect_page_type` checked `"GET /"` but text has `"GET\n/"` | Added standalone method line check, `REQUEST TYPE`, `/OPENAPI/` fallbacks |
| Method/path empty | Regex expected same-line `GET /path` | Added separate-line detection (method on line N, path on line N+1) |
| 0 params/response/errors | BingX paired tables (1-row header table + separate data table) | `_merge_paired_tables()` detects and merges consecutive pairs |
| Response table miss | BingX typo `"filed"` not `"field"` | Added `"filed"` to header match |
| Error table miss | Header says `"error msg"` not `"error message"` | Added `"msg"` to match |
| Empty trailing cells | Extra `""` column in header rows | Strip empty cells during merge |
| Click timeout on hidden items | Menu items not visible without scrolling | `scroll_into_view_if_needed()` + re-expand nav fallback |
| Stale progress resume | Test mode progress file loaded by full scrape | Test/section modes delete progress file |
